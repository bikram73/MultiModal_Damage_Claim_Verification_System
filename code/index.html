What it does

System Capabilities

Every claim is processed through a multi-layer verification pipeline combining visual analysis, text parsing, and risk scoring.

image_search

Multi-Modal Visual Analysis

Inspects submitted images against the claim. Blur, glare, wrong-angle, and text-injection are detected automatically.

chat_bubble

Conversation Parsing

Extracts the exact damage claim from the support transcript to identify object part and issue type.

history

User History Risk

Cross-references claim history to flag repeated patterns or manipulation without overriding visual evidence.

checklist

Evidence Standard Enforcement

Maps each claim to a minimum evidence checklist and reports whether the image set satisfies it.

security

Prompt Injection Detection

Detects text instructions in transcripts or images that attempt to override the verification system.

bar_chart

Dual-Strategy Engine

Strategy A: fast heuristic rules. Strategy B: VLM visual inspection — switchable from the dashboard.

How it works

Verification Pipeline

1

Load Input

Read claims.csv, user history, and evidence requirements.

2

Conversation Parse

Extract claim object, issue type, and object part from the customer utterance.

3

Image Analysis

OpenCV checks blur, glare, brightness. VLM strategy inspects actual damage.

4

Risk Scoring

User history, image quality flags, and text injection signals merged into risk score.

5

Decision Output

Produces output.csv with status, justification, severity, evidence standard, and supporting image IDs.
