What it does

System Capabilities

Every claim is processed through a multi-layer verification pipeline combining visual analysis, text parsing, and risk scoring.

image_search

Multi-Modal Visual Analysis

GPT-4o vision inspects submitted images against the claim. Blur, glare, wrong-angle, and text-injection are detected automatically.

chat_bubble

Conversation Parsing

Extracts the exact damage claim from the support transcript, focusing on the last customer statement to identify object part and issue type.

history

User History Risk Context

Cross-references claim history to flag repeated patterns, exaggerations, or manipulation attempts without overriding clear visual evidence.

checklist

Evidence Standard Enforcement

Maps each claim to a minimum evidence checklist (e.g. REQ_CAR_BODY_PANEL) and reports whether the image set satisfies it.

security

Prompt Injection Detection

Detects and flags text instructions embedded in claim transcripts or images that attempt to override the verification system.

bar_chart

Multi-Strategy Engine

Four interchangeable strategies: Heuristic (zero cost), GPT-4o vision, Gemini 2.0 Flash, and Qwen2.5-VL-72B via HuggingFace — all switchable from the dashboard.

How it works

Verification Pipeline

1

Load Input

Read claims.csv, user history, and evidence requirements. Resolve image paths to dataset/images/.

2

Conversation Parse

Extract claim object, issue type, and object part from the final customer utterance using keyword detection.

3

Image Analysis

OpenCV checks blur, glare, and brightness. GPT-4o (VLM strategy) performs visual inspection of actual damage in the submitted photos.

4

Risk Scoring

User history, image quality flags, and text injection signals are merged into risk flags and a composite risk score.

5

Decision Output

Produces output.csv with status, justification, severity, evidence standard, and supporting image IDs for each claim.
